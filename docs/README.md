## 后续文档约定

新增文档建议按主题组织，例如：

- `architecture/`：系统架构、模块边界、关键技术决策
- `api/`：接口契约、错误码和事件定义
- `development/`：本地开发、编码规范和测试策略
- `operations/`：部署、监控、备份、恢复和故障处理

单个应用内部的启动和调试说明保留在应用目录中，例如 [`../frontend/README.md`](../frontend/README.md)。
